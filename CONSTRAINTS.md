# ВСК ДМС-ассистент — Ограничения MVP

Документ фиксирует текущие технические и продуктовые ограничения версии MVP.
Каждый пункт — осознанное упрощение, а не дефект.

---

## Обработка запросов

**Синхронный блокирующий эндпоинт.**
`POST /api/update` не возвращает ответ до полной обработки файла.
На 322 строки при Gemini free tier — от 2 до 10 минут. Параллельные запросы встают в очередь uvicorn.
> Для production потребуется очередь задач (Celery, ARQ) и асинхронный статус-эндпоинт.

**Один запрос одновременно.**
Нет воркеров, нет пула. Второй запрос ждёт завершения первого.

**Нет повторной обработки при сбое.**
Если LLM вернул ошибку на одном из чанков — весь запрос падает с 500. Частичный результат не сохраняется.

---

## LLM и качество

**Качество ответа не гарантировано.**
LLM может нарушить количество строк в чанке, объединить или пропустить пункты — особенно при сложных формулировках. Проверка корректности количества строк в постпроцессоре отсутствует.

**Сопоставление строк — эвристическое, не семантическое.**
`ExcelReportWriter` ищет строку в исходном файле по трём уровням: точное совпадение → совпадение префикса → Jaccard-перекрытие слов ≥ 75 %. Парафразы и синонимы могут не совпасть — строка останется без аннотации.

**RAG по ключевым словам (Jaccard), не семантический.**
`NormativeIndex` сопоставляет разделы базы с чанком по пересечению слов. Синонимы и парафразы не учитываются. При `LLM_MAX_SECTIONS=2` в промт попадают только 2 раздела — остальные игнорируются.

**Нормативная база перечитывается при каждом запросе.**
Файл читается с диска и индексируется заново при каждом вызове `POST /api/update`. При очень большой базе (>1 МБ) это заметная лишняя работа.
> Для production — кэшировать `NormativeIndex` на уровне приложения с инвалидацией по mtime файла.

**Лимиты бесплатных API.**
- Gemini free tier: 20 запросов/день на модель. 322 строки = 13 чанков = 13 запросов, что близко к лимиту.
- Anthropic: лимиты по токенам в минуту, возможны паузы между чанками.

## Входные данные

**PDF — только текстовый, не сканы.**
Сканированные PDF (изображения) не читаются. Нужен OCR — в текущей версии не реализован.

**Структура таблицы не валидируется.**
Система ожидает, что требования клиента — это строки таблицы или нумерованные пункты. Произвольный текст без структуры может привести к некорректной разбивке на чанки.

**Файл клиента передаётся как путь внутри контейнера.**
API принимает `file_path` — путь к файлу внутри контейнера. Загрузка файла через HTTP multipart не поддерживается. GUI копирует файл в `uploads/`, примонтированный как volume.

---

## Нормативная база

**Несколько файлов в директории конкатенируются.**
`NormativeBaseLoader` читает все файлы из `NORMATIVE_BASE`-директории и объединяет их в один текст. Если при обновлении базы старый файл не удалить, оба попадут в промт одновременно.

**Нет версионирования базы.**
Нет истории изменений нормативных документов. Неизвестно, по какой версии базы был сформирован конкретный ответ.

---

## Безопасность и доступ

**Нет аутентификации.**
Любой, кто имеет доступ к порту 8001, может отправить запрос.

**Нет валидации пути к файлу.**
`file_path` в запросе не проверяется на path traversal. В production обязательна санитизация.

**API-ключи в `.env` в открытом виде.**
Секреты не вынесены в vault или secrets manager.

---

## Выходные данные

**Выходной файл перезаписывается.**
При повторной обработке `client_ответ.xlsx` перезаписывается без предупреждения.

**Для PDF и неизвестных форматов — fallback на .docx.**
Оригинальная структура PDF не сохраняется — ответ создаётся как новый Word-документ.

**Debug-файлы пишутся рядом с исходником.**
`client_debug.md` создаётся в той же папке, что и исходный файл. Нет ротации, накапливаются на диске.

## GUI-приложение

**Оценка времени — приблизительная.**
Прогресс-бар считает время как `(количество строк / 25) × 15 секунд`. Реальное время зависит от провайдера, размера нормативной базы и нагрузки на API.

**Нет отмены обработки.**
После нажатия «Подготовить» запрос нельзя прервать через GUI — только закрыть приложение (запрос на сервере продолжится).

**Приложение работает только при запущенном контейнере.**
GUI ожидает FastAPI на `http://localhost:8001`. Если контейнер не поднят, кнопка «Подготовить» вернёт ошибку подключения.

---

## Инфраструктура

**Нет мониторинга и алертов.**
Логи только в stdout контейнера. Нет метрик, нет трейсинга запросов.

**Нет истории обработки.**
Каждый запрос независим. Нет базы данных, нет логов по `request_id`.

**Один инстанс, нет горизонтального масштабирования.**
Docker Compose, один контейнер API.
